# Using an AWS E3 VM as an agent to stream live data continuously into Foundry

Optimized VM SKU config for data streaming agent:
- SKU: t3 micro (2vCPUs, 1GiB RAM) - cheapest
- Security group: SSH inbound enabled, HTTP outbound enabled
- Disk: General purpose SSD 8GiB (3000 IOPS)

On the VM:
- update packages
```
sudo apt update
```
- install python libraries: websockets and requests
```
sudo apt install -y python3-websockets python3-requests
```

Establish Connection to Foundry:
- In Foundry: create stream data set, then follow instruction to generate token or secret, I used token
- (replace token: settings --> token)
- create a python file on the VM and paste the in Foundry autogenerated script, to avoid errors, you might need to change the timestamp in the script to this: "timestamp": 1713207505000
```
nano app.py
```
- run test script
```
python3 app.py
```
- Validate in Foundry that the connection has been established:
<img width="2088" height="397" alt="image" src="https://github.com/user-attachments/assets/44265eac-44ea-4001-b204-d702942d2812" />

- paste app.py from this folder and run it on the VM
- make sure you see live data (5s refresh) in Foundry
  
Let process continue after you disconnect from the VM, using systemd:
- create .env file in home:
```
FOUNDRY_TOKEN=your-token-here
DATASET_RID=your-dataset-rid
FOUNDRY_URL=your-foundry-url
```
- create systemd service file in the system directory
```
sudo nano /etc/systemd/system/binance-stream.service
```
- copy the following content
```
[Unit]
Description=Binance to Foundry Streaming Agent
After=network.target

[Service]
Type=simple
User=ubuntu
WorkingDirectory=/home/ubuntu
EnvironmentFile=/home/ubuntu/.env
ExecStart=/usr/bin/python3 /home/ubuntu/app.py
Restart=always
RestartSec=5
StandardOutput=append:/home/ubuntu/stream.log
StandardError=append:/home/ubuntu/stream.log

[Install]
WantedBy=multi-user.target
```
